# v0.1.5

## Highlights
- File-based theming with built-in and user themes loaded from TOML.
- New `theme` CLI commands to list, preview, and set themes.
- TUI widgets now render using theme-aware colors.

## Added
- Theme parsing, validation, and fallback behavior for missing or invalid themes.
- Built-in themes: `default`, `dracula`, `catppuccin-mocha`, `nord`, `solarized-dark`.
- User config bootstrap in `~/.config/omakure/` with `themes/` and `config.toml`.
- `omakure theme` subcommands: `list`, `set`, `preview`, `path`.
- Unit tests covering theme parsing and usage notes.

## Changed
- App startup loads the configured theme before rendering.
- TUI widgets use themed colors instead of hardcoded values.
- `.opencode` directory ignored by git.
